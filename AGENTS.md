# AGENTS.md

## Language
- Always respond in Traditional Chinese.
- Keep all IT and technical terms in English (e.g., controller, model, migration, callback).
- Use English for all code.

## Rails Guidelines
- This project uses Rails 8.1.
- Follow Rails conventions where practical.
- Prefer standard Rails patterns and best practices.
- Prefer service objects for complex business logic.
- Avoid fat controllers.

## Code Style
- For newly written code, prefer Ruby and Rails community conventions.
- New code should aim to follow RuboCop style when practical.
- Do not require unrelated legacy code to be refactored only for style compliance.
- When editing legacy code, minimize unnecessary refactoring unless requested.
- Keep code clean, readable, and maintainable.

## Explanation Style
- Explain logic in Traditional Chinese.
- Be concise but practical.
- Provide examples when helpful.
